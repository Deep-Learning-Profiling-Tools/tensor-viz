# Agent Instructions

- After making code changes,
  run `npm run build`
  from the repository root
  before considering the task complete.

- Do not treat workspace-level test commands
  as a substitute for `npm run build`;
  the built demo assets must be refreshed.

- When adding code comments,
  explain why the code is needed,
  not merely what it does.

- Good comments should name the failure mode:
  what inputs, states, or event orderings break without the code,
  and how the code avoids or fixes that bug.

- Avoid comments that just restate the implementation.
